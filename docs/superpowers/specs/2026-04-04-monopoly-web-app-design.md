# モノポリ Web アプリ 設計ドキュメント

## 概要

スマートフォン1台で複数人が交代で遊べるモノポリWebアプリ。最大4人対応（人数変更可能）。将来的にはオンライン対戦に拡張予定だが、初期バージョンは1台のスマートフォンを手渡しして遊ぶローカルプレイのみ。

**ターゲットユーザー**: 家族（小学生の子どもを含む）

## 技術スタック

| カテゴリ       | 選定                  | 理由                                              |
| -------------- | --------------------- | ------------------------------------------------- |
| ビルドツール   | Vite                  | 高速、React対応、設定シンプル                     |
| フレームワーク | React 19 + TypeScript | 型安全、要件通り                                  |
| スタイリング   | CSS Modules           | コンポーネント単位のスコープ、追加依存なし        |
| 状態管理       | useReducer            | 純粋関数でゲームロジック分離                      |
| サウンド       | Howler.js             | Web Audio APIのラッパー、軽量、ブラウザ互換性良好 |
| テスト         | Vitest                | Viteと統合、ゲームロジックのユニットテスト        |
| リンター       | ESLint + Prettier     | コード品質                                        |

## アーキテクチャ

### ディレクトリ構成

```
src/
├── components/          # UIコンポーネント
│   ├── Board/           # ボード表示（フォーカス+ミニマップ）
│   ├── Dice/            # サイコロアニメーション
│   ├── PlayerPanel/     # プレイヤー情報パネル
│   ├── PropertyCard/    # 物件カード表示
│   ├── ActionDialog/    # 購入・交渉等のダイアログ
│   └── Setup/           # ゲーム開始設定画面
├── game/                # ゲームロジック（純粋関数、UIに依存しない）
│   ├── reducer.ts       # メインのgameReducer
│   ├── actions.ts       # アクション定義
│   ├── board.ts         # ボードデータ（40マス定義）
│   ├── cards.ts         # Chance/Community Chestカード
│   ├── rules.ts         # ルール判定ロジック
│   └── types.ts         # 型定義
├── sound/               # サウンド管理
│   └── useSoundEffect.ts
├── App.tsx
└── main.tsx
```

**設計原則:**

- `game/`ディレクトリはUIに一切依存しない純粋なロジック層。テスト容易性と将来のオンライン対応を見据えた分離
- `components/`は見た目とユーザー操作のみ担当
- 状態は`App.tsx`の`useReducer`で一元管理し、各コンポーネントにpropsで渡す

## データ構造

### ゲーム状態

```typescript
type GameState = {
  phase: 'setup' | 'playing' | 'finished'
  players: Player[]
  currentPlayerIndex: number
  board: BoardSpace[]
  cards: { chance: Card[]; communityChest: Card[] }
  dice: { values: [number, number]; doubles: number }
  turnPhase: 'roll' | 'moving' | 'landed' | 'action' | 'endTurn'
}
```

### プレイヤー

```typescript
type Player = {
  id: string
  name: string
  token: string // 絵文字 🚗🎩👞🐕
  money: number // 初期$1500
  position: number // 0-39
  properties: string[] // 所有物件ID
  inJail: boolean
  jailTurns: number
  getOutOfJailCards: number
  isBankrupt: boolean
}
```

### ボードマス

```typescript
type BoardSpace = {
  id: string
  position: number // 0-39
  type:
    | 'property'
    | 'railroad'
    | 'utility'
    | 'tax'
    | 'chance'
    | 'communityChest'
    | 'corner'
  name: string
  color?: string // 物件カラーグループ
  price?: number
  rent?: number[] // [基本, 1軒, 2軒, 3軒, 4軒, ホテル]
  houseCost?: number
  owner?: string // プレイヤーID
  houses: number // 0-4 (5=ホテル)
  isMortgaged: boolean
}
```

### カード

```typescript
type Card = {
  id: string
  type: 'chance' | 'communityChest'
  text: string // カードに表示するテキスト（子ども向けのやさしい日本語）
  action:
    | { type: 'move'; position: number } // 指定マスへ移動
    | { type: 'moveRelative'; spaces: number } // 相対移動
    | { type: 'money'; amount: number } // お金を受け取る/支払う（負の値は支払い）
    | { type: 'moneyFromPlayers'; amount: number } // 各プレイヤーから受け取る
    | { type: 'jail' } // 刑務所へ行く
    | { type: 'jailFree' } // 刑務所から出られるカードを手に入れる
    | { type: 'repair'; perHouse: number; perHotel: number } // 家/ホテルごとに支払い
    | { type: 'moveNearest'; spaceType: 'railroad' | 'utility' } // 最寄りの鉄道/公共事業へ
}
```

### 競売（オークション）フロー

プレイヤーが物件を購入しない場合、競売が発生する:

1. 銀行が物件を$1から競売に出す
2. 現在のプレイヤーから時計回りに、入札するかパスするかを選択
3. 入札額は前の入札額より$1以上高くする必要がある
4. 全員がパスしたら、最後の入札者が落札
5. 誰も入札しなかった場合、物件は銀行に残る

UIでは各プレイヤーに「+$10」「+$50」「+$100」のボタンとパスボタンを表示し、テンポよく進められるようにする。

## ゲームフロー

### ターンの流れ

```
サイコロを振る → 移動アニメーション → マスに着地 → アクション実行 → ターン終了
     ↑                                                              |
     └─── ゾロ目の場合、もう一度 ────────────────────────────────────┘
```

`turnPhase`で状態遷移を管理: `roll` → `moving` → `landed` → `action` → `endTurn`

### 各マスタイプのアクション

| マスタイプ      | 未所有時             | 他プレイヤー所有時             | 自分所有時 |
| --------------- | -------------------- | ------------------------------ | ---------- |
| 物件            | 購入 or 競売         | レンタル支払い                 | 何もなし   |
| 鉄道            | 購入 or 競売         | レンタル支払い（所有数で変動） | 何もなし   |
| 公共事業        | 購入 or 競売         | サイコロ×倍率支払い            | 何もなし   |
| 税金            | 所定額を銀行に支払い | -                              | -          |
| Chance          | カードを引く         | -                              | -          |
| Community Chest | カードを引く         | -                              | -          |
| GO              | $200受け取り         | -                              | -          |
| 刑務所（訪問）  | 何もなし             | -                              | -          |
| Go to Jail      | 刑務所へ移動         | -                              | -          |
| Free Parking    | 何もなし             | -                              | -          |

### 交渉・取引

- 自分のターン中いつでも交渉ボタンから開始
- 物件・お金・Get Out of Jail Freeカードを交換可能
- 両プレイヤーが同意で成立

### 家・ホテルの建設ルール

- カラーグループ全物件を所有時に建設可能
- 均等建設ルール（同グループ内の家の差は1以内）
- 家4軒→ホテルにアップグレード
- 抵当中の物件があるグループには建設不可

### 破産処理

- 支払い不能時、まず家/ホテル売却（半額）→ 抵当設定 → それでも不足なら破産
- 破産先が他プレイヤーの場合：全資産がそのプレイヤーへ
- 破産先が銀行の場合：全物件を競売

### ビジュアルスタイル: ポップ/カジュアル

- 明るい配色、丸みのあるUI、カジュアルゲーム風
- 絵文字アイコンのプレイヤートークン（🚗🎩👞🐕）
- ターン切り替えはシンプル（情報オープン、プライバシー保護なし）

### 子ども向けのUI配慮

- **大きなボタン**: タップしやすい大きめのボタン
- **やさしい日本語**: 難しい用語は言い換え
- **色でわかる**: お金が増えたら緑、減ったら赤のフラッシュ
- **ガイド表示**: 「このマスは○○さんのだよ。$50はらってね！」のような説明
- **確認ダイアログ**: 重要な操作は「ほんとうにいい？」と確認

## アニメーション

- **サイコロ**: CSS animationで2つのサイコロが回転→結果表示（約1.5秒）
- **駒の移動**: マスを1つずつ進むアニメーション（1マス約0.3秒）
- **お金の増減**: 数字がカウントアップ/ダウンする演出
- **カード**: 下からスライドインして表示
- **家/ホテル**: ポンッと出現するバウンスアニメーション

## レスポンシブ対応

- スマートフォン縦画面（375px〜430px）を最優先
- `viewport meta`でモバイル最適化
- タッチ操作前提（ホバー状態なし）
- 横画面はロックまたは警告表示

## ブラウザ対応

- iOS Safari、Android Chrome を最優先
- PWA対応は将来検討（今回はスコープ外）
